# Prescripto - Doctor Booking & Healthcare Management System

A modern, full-stack MERN (MongoDB, Express, React, Node.js) application designed for seamless doctor appointment booking, prescription management, and clinic administration. It features a patient portal, a dedicated admin/doctor dashboard, a real-time AI Chatbot, and online payments.

---

## 🚀 Key Features

*   **User/Patient Portal**:
    *   Secure Authentication with a global Route Guard.
    *   Search and filter doctors by specialities (General Physician, Gynecologist, Dermatologist, Pediatrician, Neurologist, Gastroenterologist).
    *   Flexible appointment booking with slot selection.
    *   Multiple Payment Gateways: Pay at Clinic (Cash) & Online (integrated securely via **Stripe**).
    *   Interactive **AI Chatbot** powered by **Gemini 2.5 Flash** for symptom checks and medical recommendations.
    *   Downloadable/printable digital prescriptions generated by doctors.
*   **Admin Dashboard**:
    *   Comprehensive analytics (total doctors, appointments, patients, recent bookings).
    *   Manage doctors (add, edit experience, edit bio/about, update availability, or delete).
    *   View all patient appointments and update payment/cancellation states.
*   **Doctor Dashboard**:
    *   Manage scheduled appointments (mark completed, cancel).
    *   Add digital prescriptions directly to patients' appointments.
    *   Monitor earnings and individual profile.

---

## 🛠️ Tech Stack

*   **Frontend & Admin**: React.js (Vite), Tailwind CSS, React Router DOM, Axios, React Toastify.
*   **Backend**: Node.js, Express, MongoDB (Mongoose ODM).
*   **Third-Party Integrations**:
    *   **Cloudinary**: For hosting doctor/user profile pictures.
    *   **Stripe**: Secure online payment gateway (with dynamic redirection).
    *   **Google Gemini AI**: Intelligent natural language processing for the virtual assistant.

## ⚙️ Environment Variables Setup

### 1. Backend (`Backend/.env`)
Create a `.env` file inside the `Backend` directory:
```env
PORT=4000
MONGO_URI=your_mongodb_connection_string
CLOUDINARY_NAME=your_cloudinary_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
ADMIN_EMAIL=admin@prescripto.com
ADMIN_PASSWORD=your_admin_password
JWT_SECRET=your_jwt_secret_token
STRIPE_SECRET_KEY=your_stripe_secret_key
GEMINI_API_KEY=your_gemini_api_key
```

### 2. Patient Portal (`frontend/.env`)
Create a `.env` file inside the `frontend` directory:
```env
VITE_BACKEND_URL='http://localhost:4000' # Change to production URL when live
```

### 3. Admin Portal (`admin/.env`)
Create a `.env` file inside the `admin` directory:
```env
VITE_BACKEND_URL='http://localhost:4000' # Change to production URL when live
```

## 🚀 Getting Started

### 1. Database Initialization (Seeding)
To pre-populate the database with the required **16 doctor profiles** containing pre-uploaded Cloudinary images:
```bash
cd Backend
npm install
node seed.js
```

### 2. Run the Backend
```bash
npm run server
```
*Runs the server at `http://localhost:4000` using `nodemon` (auto-reloads on file changes).*

### 3. Run the Frontend Client
```bash
cd ../frontend
npm install
npm run dev
```
*Runs the patient website on `http://localhost:5173`.*

### 4. Run the Admin/Doctor Console
```bash
cd ../admin
npm install
npm run dev
```
*Runs the admin dashboard on `http://localhost:5174` (or next available port).*

## 🌐 Production Deployment

### Backend
1. Deploy to a platform like **Render**, **Railway**, or **AWS**.
2. Add all environmental variables from `Backend/.env` to the platform's Environment Settings.
3. Make sure to whitelist the IP addresses if your MongoDB Cluster restricts access.

### Frontend & Admin
1. Deploy both repositories as separate static sites on **Vercel** or **Netlify**.
2. For both projects, set the root directory to `frontend` and `admin` respectively.
3. Set the build command to `npm run build` and output directory to `dist`.
4. Add the environment variable `VITE_BACKEND_URL` pointing to the live URL of your deployed backend (e.g. `https://your-api.onrender.com`).
